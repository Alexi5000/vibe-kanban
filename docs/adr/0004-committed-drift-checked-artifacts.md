# ADR 0004: Commit contract and benchmark artifacts, enforce with drift checks

Status: Accepted (2026-07-06)

## Context

Generated artifacts (JSON Schema contracts, eval summaries) can silently
diverge from their sources. Regenerating them on every consumer's machine
hides that divergence; committing them without verification lets them rot.

## Decision

Both artifacts are committed **and** re-derived in CI:

- `contracts/schema.json` + `contracts/schema.sha256` — regenerated by
  `contracts:sync`; `contracts:check` fails if the committed bytes, the hash,
  or a fresh build disagree.
- `docs/benchmark/offline-eval-summary.json` — regenerated by `fde:evals`;
  `benchmark:check` fails if the committed summary differs from a fresh run
  or if the human-readable table in `docs/BENCHMARK.md` no longer matches the
  summary's rows.

Determinism makes this possible: artifacts carry `"generatedAt":
"deterministic"` instead of timestamps.

## Consequences

- Reviewers see contract and benchmark changes as ordinary diffs.
- A PR cannot change harness behavior without the benchmark diff showing it.
- The pattern requires strict determinism discipline in the harness (no
  randomness, no clocks, stable orderings).
